# 测试多用户加入婚礼功能指南

## 测试步骤

### 第一步：获取当前用户的婚礼ID

1. **使用当前用户登录**（如果还没登录）
   - 访问 http://localhost:3000/login
   - 使用现有账号登录

2. **查看婚礼详情页获取短ID**
   - 访问 http://localhost:3000/weddings
   - 点击任意一个婚礼进入详情页
   - 在"邀请他人加入"部分，你会看到6位数字的婚礼ID（例如：123456）
   - **复制这个ID**，稍后会用到

### 第二步：创建测试用户账号

1. **退出当前账号**
   - 点击页面上的"退出"按钮，或访问 http://localhost:3000/api/auth/logout

2. **注册新用户**
   - 访问 http://localhost:3000/register
   - 填写注册信息：
     - 用户名：`testuser`（或其他你喜欢的名字）
     - 邮箱：`test@example.com`（可选）
     - 密码：`123456`（至少6位）
   - 点击"注册"
   - 注册成功后会自动登录并跳转到婚礼列表页

### 第三步：使用新用户加入婚礼

1. **访问加入页面**
   - 点击页面上的"加入婚礼"按钮，或直接访问 http://localhost:3000/weddings/join

2. **输入婚礼ID**
   - 在输入框中输入第一步复制的6位数字ID（例如：123456）
   - 点击"查询"按钮预览婚礼信息

3. **确认并加入**
   - 查看显示的婚礼信息（名称、日期、创建人等）
   - 点击"加入婚礼"按钮
   - 系统会自动将你添加为参与者

4. **验证加入成功**
   - 加入成功后会自动跳转到婚礼详情页
   - 你应该能看到婚礼的所有任务和详情
   - 在"我的婚礼"页面（http://localhost:3000/weddings）也应该能看到这个婚礼

## 测试检查点

✅ **功能测试**
- [ ] 新用户可以通过6位数字ID成功加入婚礼
- [ ] 加入后可以在"我的婚礼"列表中看到该婚礼
- [ ] 加入后可以查看婚礼的所有任务
- [ ] 加入后可以更新任务状态（如果是参与者）

✅ **边界情况测试**

✅ **UI/UX测试**
- [ ] 加入页面的输入框只接受数字
- [ ] 查询功能可以预览婚礼信息
- [ ] 复制ID功能正常工作
